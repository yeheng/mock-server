# WireMock UI 使用指南

## 快速开始

### 1. 启动开发服务器
```bash
cd ui
npm run dev
# 或者使用提供的启动脚本
./start.sh
```

### 2. 访问界面
打开浏览器访问: http://localhost:5173

## 界面功能介绍

### 📊 仪表板
- **概览**: 查看所有stub的统计信息
- **快速操作**: 快速执行常用操作
- **活动日志**: 了解系统中的最新变更
- **最近Stub**: 快速访问最新创建的stub

### 📋 Stub列表
- **浏览**: 查看所有stub的列表
- **搜索**: 按名称或URL搜索特定stub
- **筛选**: 实时筛选和排序
- **批量操作**: 选择多个stub进行批量操作
- **分页**: 高效浏览大量数据

### ➕ 创建/编辑Stub
- **基本信息**: 设置名称、描述、HTTP方法、URL
- **请求匹配**: 定义哪些请求应该匹配此stub
- **响应配置**: 设置返回的状态码、响应头和响应体
- **验证**: 实时验证表单输入的完整性

### 🔍 查看Stub详情
- **完整信息**: 查看stub的所有配置详情
- **测试功能**: 直接测试stub的响应
- **JSON查看**: 查看和复制原始配置

## 操作流程示例

### 创建新的Stub映射
1. 点击左侧导航的"创建 Stub"或仪表板的"创建新 Stub"按钮
2. 填写基本信息：
   - 名称: "用户登录接口"
   - 方法: POST
   - URL: `/api/login`
   - 描述: "用户登录认证接口"
3. 配置请求匹配（可选）：
   - 请求头: `Content-Type: application/json`
   - 请求体: 检查用户凭据
4. 配置响应：
   - 状态码: 200
   - 响应头: `Content-Type: application/json`
   - 响应体: 返回用户token
5. 点击"创建"保存

### 批量管理Stub
1. 在Stub列表中勾选要操作的stub
2. 使用批量操作按钮：
   - 批量启用: 启用所有选中的stub
   - 批量禁用: 禁用所有选中的stub
   - 批量删除: 删除所有选中的stub（谨慎操作）

### 测试Stub配置
1. 点击stub行中的"查看"按钮
2. 在详情页面点击"测试"按钮
3. 系统将发送模拟请求并显示结果
4. 检查响应状态码、响应头和响应体内容

## 键盘快捷键

- `Ctrl/Cmd + F`: 快速搜索
- `Ctrl/Cmd + N`: 创建新stub
- `Escape`: 关闭模态框
- `Tab`: 在表单字段间切换

## 最佳实践

### Stub命名规范
- 使用描述性的名称，如"用户登录接口"而非"api1"
- 包含HTTP方法，如"[POST] 用户登录接口"
- 描述功能目的，便于团队协作

### URL模式组织
- 使用RESTful风格：`/api/users/{id}`
- 合理使用路径参数和查询参数
- 保持URL结构的一致性

### 响应体设计
- 使用JSON格式返回数据
- 包含合适的HTTP状态码
- 添加必要的响应头信息

### 版本控制
- 定期导出stub配置
- 在Git中保存配置变更
- 使用描述性的提交信息

## 故障排除

### 常见问题

**Q: 页面无法访问**
A: 检查开发服务器是否启动，端口5173是否被占用

**Q: API调用失败**
A: 确认后端WireMock服务是否正常运行，检查网络连接

**Q: 表单提交失败**
A: 检查必填字段是否已填写，验证HTTP方法是否支持

**Q: 样式显示异常**
A: 清除浏览器缓存，重新构建项目

### 调试模式
在浏览器控制台中设置：
```javascript
// 启用详细日志
localStorage.setItem('wiremock-debug', 'true')
```
